title: "Send Your First Email with Mailchimp Transactional"

---

Mailchimp Transactional is a delivery API built for triggered, one-to-one email — order confirmations, password resets, shipping notifications, and more. This page covers everything you need to go from a blank slate to a successfully delivered message: generating an API key, authenticating your sending domain, firing off a test email, and interpreting the response.

<Steps>

  <Step title="Get your API key">
    Every request to the Mailchimp Transactional API must include a valid API key. To generate one:

    1. Log in to your account at [https://mandrillapp.com/](https://mandrillapp.com/).
    2. Open **Settings** from the left navigation.
    3. Click **+ New API Key**.
    4. Give the key a descriptive label, then copy the generated value and store it somewhere safe.

    <Warning>
      Treat your API key like a password. Never commit it to source control or expose it in client-side code.
    </Warning>
  </Step>

  <Step title="Authenticate your domain">
    You must add and verify a sending domain before Mailchimp Transactional can deliver email on your behalf. Without a verified domain, messages will be rejected.

    1. Navigate to **Sending Domains** in the dashboard.
    2. Click **Add Domain** and enter the domain you plan to send from.
    3. Follow the DNS setup instructions — at minimum, you must publish the **DKIM record** provided in the dashboard.
    4. Once your DNS changes propagate, click **Test DNS Settings** to confirm verification.

    <Tip>
      Use a subdomain like `mail.yourdomain.com` to keep transactional and marketing email reputation separate.
    </Tip>
  </Step>

  <Step title="Send a test message">
    With a key in hand and a verified domain, you're ready to send. Make a `POST` request to the `/messages/send` endpoint:

    ```bash
    curl -X POST https://mandrillapp.com/api/1.0/messages/send \
      -H "Content-Type: application/json" \
      -d '{
        "key": "YOUR_API_KEY",
        "message": {
          "from_email": "orders@yourdomain.com",
          "from_name": "Your Company",
          "subject": "Your order has shipped",
          "html": "<p>Hi Jordan, your order #1043 has shipped and is on the way.</p>",
          "to": [
            { "email": "jordan@example.com", "name": "Jordan Lee", "type": "to" }
          ],
          "track_opens": true,
          "track_clicks": true
        }
      }'
    ```

    Replace `YOUR_API_KEY` with the key you copied in Step 1, and replace `orders@yourdomain.com` with an address on your verified domain.
  </Step>

  <Step title="Check the response">
    A successful call returns a JSON array with one object per recipient:

    ```json
    [
      {
        "email": "jordan@example.com",
        "status": "sent",
        "_id": "abc123abc123abc123abc123",
        "reject_reason": null
      }
    ]
    ```

    - A `status` of `sent` or `queued` means delivery is underway.
    - A `status` of `rejected` or `invalid` means the message was not sent — check the `reject_reason` field for details on why it was blocked.
    - Save the `_id` value to look up delivery status, open events, and click events for that message later.
  </Step>

</Steps>
