Hardcoding HTML in every API call gets unwieldy fast. Templates let your team manage email content separately from your application code, and merge variables personalize each send.

## Why templates

## Sending with a template

Use the [`/messages/send-template`](/api-reference/messages/send-template) endpoint and reference a template by its slug:

```json
{
  "key": "YOUR_API_KEY",
  "template_name": "order-confirmation",
  "template_content": [],
  "message": {
    "from_email": "orders@example.com",
    "subject": "Your order is confirmed",
    "to": [{ "email": "jordan@example.com", "type": "to" }],
    "merge_vars": [
      {
        "rcpt": "jordan@example.com",
        "vars": [
          { "name": "ORDER_ID", "content": "1043" },
          { "name": "TOTAL", "content": "$48.00" }
        ]
      }
    ]
  }
}
```

## Merge variables

Reference variables in your template with merge tags like `*|ORDER_ID|*`. At send time, each recipient's `vars` replace the tags. Use `global_merge_vars` for values shared across all recipients and `merge_vars` for per-recipient values.

<Note>
  Set `merge_language` to `handlebars` if you prefer Handlebars syntax over the default Mailchimp merge tags.
</Note>
