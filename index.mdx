Mailchimp Transactional gives you a powerful, scalable API for sending one-to-one triggered emails and SMS messages — think password resets, order confirmations, shipping notifications, and other time-sensitive communications. This page introduces the platform, explains how it fits into your application, and points you to everything you need to get started quickly.

<Note>
  Mailchimp Transactional was formerly known as Mandrill. You may see references to both names in the dashboard and API responses.
</Note>

## How it works

<Steps>
  <Step title="Get an API key">
    Log in to [mandrillapp.com](https://mandrillapp.com), open **Settings**, and click **New API Key**. Copy the key — you'll include it in every request you make to the API.
  </Step>
  <Step title="Send a message">
    Call `POST /messages/send` with your API key and a message object that includes your recipients, subject, and content. The API returns a status for each recipient immediately.
  </Step>
  <Step title="Track delivery">
    Check the status of any message by querying the API, or configure webhooks to receive real-time events — opens, clicks, bounces, and more — delivered directly to your server.
  </Step>
</Steps>
